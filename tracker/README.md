# Laptop Tracker

Organisation-level laptop tracking system with role-based access, live device maps, and location history replay.

## Architecture

```
tracker/
├── backend/        FastAPI (Python) — REST API + JWT auth
├── dashboard/      React + Vite + Leaflet — web dashboard
├── agent/          Electron — tray app that runs on tracked laptops
└── render.yaml     One-click Render deployment config
```

## Roles

| Role | Access |
|------|--------|
| `super` | All organisations, all users, can create orgs and managers |
| `manager` | Their organisation only — map, user list, register users |
| `user` | Their own device location and history only |

## Render deployment (backend + dashboard)

1. Push this repo to GitHub
2. In Render dashboard → New → Blueprint → select your repo
3. Render reads `render.yaml` and creates all three services automatically
4. After first deploy, enable PostGIS:
   ```
   Render shell → psql $DATABASE_URL -c "CREATE EXTENSION IF NOT EXISTS postgis;"
   ```
5. Create the first super user via the Render shell:
   ```python
   cd backend
   python -c "
   from app.core.database import SessionLocal
   from app.models.user import User, UserRole
   from app.core.auth import hash_password
   db = SessionLocal()
   u = User(email='admin@yourorg.com', full_name='Admin', hashed_password=hash_password('changeme'), role=UserRole.super_user)
   db.add(u); db.commit()
   print('Super user created')
   "
   ```

## Local development

### Backend
```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
DATABASE_URL=postgresql://localhost/tracker SECRET_KEY=dev uvicorn app.main:app --reload
# API docs at http://localhost:8000/docs
```

### Dashboard
```bash
cd dashboard
npm install
VITE_API_URL=http://localhost:8000 npm run dev
```

### Agent
```bash
cd agent
npm install
API_URL=http://localhost:8000 npm start
```

## Agent distribution

Build installers for each platform:
```bash
cd agent
npm install electron-builder --save-dev
npm run build:win    # → dist/Laptop Tracker Setup.exe
npm run build:mac    # → dist/Laptop Tracker.dmg
npm run build:linux  # → dist/Laptop Tracker.AppImage
```

Distribute the installer to each employee. On first launch they log in with the credentials created by the super admin or their org manager. The app then runs silently in the system tray and posts location every 5 minutes.

## Environment variables

| Variable | Service | Description |
|----------|---------|-------------|
| `DATABASE_URL` | backend | Postgres connection string (auto-set by Render) |
| `SECRET_KEY` | backend | JWT signing secret (auto-generated by Render) |
| `VITE_API_URL` | dashboard | Backend URL (auto-set by Render) |
| `API_URL` | agent | Backend URL — set before building installers |

## API endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| POST | `/auth/login` | public | Get JWT token |
| GET | `/users/me` | any | Current user profile |
| POST | `/users/` | manager+ | Register a new user |
| GET | `/organisations/` | super | List all orgs |
| POST | `/organisations/` | super | Create org |
| GET | `/organisations/{id}/users` | manager+ | Org user list |
| POST | `/locations/` | any | Post device location (agent) |
| GET | `/locations/map/{org_id}` | manager+ | All users + latest location |
| GET | `/locations/history/{user_id}` | any* | Location history for replay |

*users can only query their own history
